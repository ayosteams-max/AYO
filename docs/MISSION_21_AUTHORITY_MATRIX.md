# Mission 21 — Authority Matrix

Status: **Architecture proposal only.**

| Capability | AI | Support human | Specialist/owning authority |
|---|---|---|---|
| Intent, summary, translation | Propose | Correct/approve | Language specialist for high-risk uncertainty |
| Case lifecycle | No authority | Commands within role | Orchestrator validates policy/state |
| Ride/arrival/pickup evidence | Cite scoped projection | Review | Owning ride/Mission 20 domains are authoritative |
| Safety/emergency | Detect and immediately route | Preserve/handoff | Safety specialist and verified local procedure own response |
| Identity/account recovery | Generic guidance only | Route | Identity/Recovery owns verification and access |
| Fraud/restriction/deactivation | Flag typed concern only | Route/appeal intake | Fraud/authorized account authority decides |
| Fare/wait dispute | Explain authoritative evidence | Investigate | Pricing determines calculation; Support/Recovery govern review |
| Refund/credit/compensation | Prohibited | Cannot move value | Future Recovery may recommend; approved financial authority executes |
| Wallet/ledger/payout | Read masked status only if authorized | Route | Ledger/payment/payout authority exclusively executes |
| Lost property | Propose contact workflow | Manage exceptions | Safety/Identity handles dangerous items or uncertainty |
| Legal/regulatory disclosure | Prohibited conclusion/action | Preserve request | Legal/privacy authority decides |
| Low-risk allow-list | Recommend candidate | Supervise/override | CTO/CEO must separately approve each executable tool/policy |
| Voice/video/co-browse channel | Language assistance only after approval | Control session within explicit grant | Identity, Safety and channel policy remain authoritative |
| Live support location | No sharing authority | Request/use only within purpose grant | User consent/lawful Safety boundary and owning location domain control |
| Family/diaspora participation | Summarize scoped context | Communicate within participant grant | Identity/authorization owns roles; Pricing/Ledger retain payer/money authority |
| Quality/CSAT scoring | May propose non-consequential rubric label | Review service quality | Cannot determine blame, restriction, employment or financial outcome |
| Knowledge base | Retrieve cited active version | Correct/escalate conflicts | Policy owner approves meaning, effective dates and withdrawal |
| Learning from resolutions | Prohibited by default | Human review alone grants no training right | Separate CTO/CEO, Privacy/Legal and AI-governance approval required |

AI confidence never expands a row’s authority.
