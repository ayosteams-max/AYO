# Mission 23 — Driver and Rider Fairness Framework

Driver: no hidden decline/cancellation punishment; no protected/irrelevant inputs;
bounded offer frequency; avoid repeat low-value loops; approved break/destination/fatigue
constraints; reason categories; appeal evidence; opportunity monitoring among materially
equivalent candidates. Fairness is not strict equal distribution.

Rider: tail-wait aging, bounded expansion, stable assignment, fake-availability rejection,
ETA freshness, no discriminatory denial/Premium downgrade, protected scheduled commitment,
dispatch-caused cancellation review and explained no-service outcomes.

Audit disparities by zone/service with minimum sample/privacy rules. Differences trigger
investigation, never automatic policy, blame or account action.
