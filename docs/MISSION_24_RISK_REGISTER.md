# Mission 24 — Risk Register

| Risk | L/I | Mitigation | Approval owner |
|---|---|---|---|
| False approval/denial | M/Critical | Multi-source deterministic policy, human review, appeal | Identity/Ops |
| SIM/phone dependence excludes users | H/High | Alternative approved methods/assisted recovery | CEO/Ops |
| Biometric bias/privacy | M/Critical | Separate gate, local evaluation, minimisation, redress | Legal/CTO |
| AI becomes verifier | M/Critical | Schema-only advice and forbidden action contracts | CTO |
| Document/provider outage | H/High | Pending/manual fallback; never auto-approve | Ops |
| Recovery becomes takeover path | H/Critical | Step-up, revoke, dual control and hold | Security |
| Trusted badge misleads riders | M/High | Purpose/expiry-specific public attributes | Product/Safety |
| Business/family overreach | M/High | Typed grants and consent | Identity/Legal |
| Excess retention/cross-border leak | M/Critical | Retention classes and provider/legal gate | Privacy/Legal |
| Insider evidence access | M/Critical | Case RBAC, audit, separation and monitoring | Security |

No critical risk is accepted for implementation.
